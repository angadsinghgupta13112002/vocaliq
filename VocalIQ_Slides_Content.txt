SLIDE 2 — THE PROBLEM
─────────────────────────────────────────────────────────
Title: The Problem

• Most people never get real feedback on how they speak
• Public speaking anxiety affects 75% of the population
• Human speaking coaches cost $100–$300/hr
• No accessible, instant, personalized tool existed

"You can't improve what you can't measure."

SLIDE 3 — THE SOLUTION
─────────────────────────────────────────────────────────
Title: VocalIQ — Your AI Speaking Coach

• Record or upload a speech → Get a full coaching report in under 90 seconds
• Powered by Google Gemini 2.5 Flash
• Scores: Content, Delivery, Confidence, Pacing, Engagement
• Timestamped transcript + improvement exercises + practice plan
• Works for: job interviews, presentations, sales pitches, lectures

SLIDE 6 — KEY FEATURE: TWO-PASS AI PIPELINE
─────────────────────────────────────────────────────────
Title: Two-Pass Gemini Coaching Pipeline

PASS 1 — Assessment
  Input:  Video/audio file (via Gemini File API)
  Output: Overall score, 5 sub-scores, emotion, WPM,
          transcript with timestamps, improvement tips

PASS 2 — Deep Coaching
  Input:  Pass 1 JSON results as context
  Output: Detailed tip cards with exercises,
          vocal control breakdown, practice plan

Why two passes?
  → Pass 1 grounds Pass 2 in facts → better, more specific coaching
  → Avoids hallucinated generic advice

─────────────────────────────────────────────────────────
SLIDE 7 — GOOGLE AUTHENTICATION
─────────────────────────────────────────────────────────
Title: Authentication — Google OAuth 2.0 + JWT

Flow:
  1. User clicks "Sign in with Google"
  2. Server redirects to Google consent screen
  3. Google returns auth code → server exchanges for user info
  4. Server upserts user doc in Firestore
  5. Server issues signed JWT (7-day expiry)
  6. Client stores JWT in localStorage
  7. All API requests send "Authorization: Bearer <token>"

Security:
  • Access token NOT stored (not needed server-side)
  • JWT verified by authMiddleware on every protected route

SLIDE 8 — GOOGLE DATABASE (FIRESTORE)
─────────────────────────────────────────────────────────
Title: Database — Cloud Firestore

Two collections:

users/
  └── {uid}
        uid, displayName, email, photoURL,
        provider, updatedAt

coaching_sessions/
  └── {uid}_{timestamp}
        userId, sessionId, mediaUrl, mode, context,
        language, overallScore, scoreBreakdown,
        transcript, improvementTips, detailedTips,
        vocalControl, strengthsToKeep, practicePlan,
        createdAt

Access model: All reads/writes via server only (JWT-protected API)
Never exposed to client directly

SLIDE 10 — ANALYTICS + LOGGING

Frontend events (gtag.js):

SLIDE 11 — DEPLOYMENT
─────────────────────────────────────────────────────────
Title: Deployment — Google Cloud Run

• Multi-stage Docker build:
  Stage 1: node:20-alpine → build React app (npm run build)
  Stage 2: node:20-alpine → copy client/dist + server, run Express

• Cloud Run serves React static files AND the API on same origin
  → /api/*  routes → Express handlers
  → everything else → React index.html (client-side routing)

• Application Default Credentials (ADC):
  → No service account key file in Docker image
  → Cloud Run service account has Firestore + GCS + Gemini roles

• Live URL: https://vocaliq-956080638663.us-central1.run.app

SLIDE 12 — LESSONS LEARNED
─────────────────────────────────────────────────────────
Title: Lessons Learned

• Express 5 breaking change: app.get("*") → must use app.use()
• Cloud Run reserves PORT — can't set it as env var
• ADC vs key files: never commit secrets; use ADC on Cloud Run
• Vite bakes API URLs at build time — need .env.production
• GA4 backend events need Measurement Protocol (not gtag.js)
• Two-pass Gemini prompting produces dramatically better coaching
