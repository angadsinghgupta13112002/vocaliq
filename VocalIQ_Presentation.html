Google Cloud Vision solution

VocalIQ does not use the standalone Cloud Vision API — and this was an intentional architectural decision.

  • Why no Cloud Vision? — Gemini 2.5 Flash is a natively multimodal model. It processes video frames, audio waveforms, and text all in a single API call — making a separate Cloud Vision integration redundant.
  • What Gemini sees instead of Vision: Full video frames (body language, facial expressions, eye contact cues) via the Gemini File API
  • What Cloud Vision would have added: Standalone image labeling or OCR — neither of which is relevant to speech coaching
  • Result: One API call to Gemini replaces what would have required Cloud Vision + Speech-to-Text + separate sentiment analysis — simpler, cheaper, and more accurate for this use case
💡 While presenting: Show the Gemini File API call in geminiCoachingService.js — specifically the part where the video file is passed as a Part to the model and Gemini returns both visual and audio analysis in one response.

Google Gemini solution

Gemini 2.5 Flash is the core AI engine of VocalIQ — used via a two-pass chained pipeline.

Pass 1 — Speech Assessment

  • Video/audio uploaded via Gemini File API
  • Returns overall score (0–10)
  • 5 sub-scores (content, delivery, confidence, pacing, engagement)
  • Detected emotion vs needed emotion
  • Words per minute estimate
  • Timestamped full transcript
  • Top improvement areas

Pass 2 — Deep Coaching

  • Pass 1 JSON fed back as context
  • Detailed tip cards (title + exercise + rephrasing)
  • Vocal control breakdown
  • Pacing: current WPM → target WPM
  • Tone, breath, pause technique
  • Strengths to keep
  • 7-day personalized practice plan

Summary

⚠ Problems We Hit

Express 5 wildcard route crash · Cloud Run reserved PORT env var · ADC vs key file crash on startup · Vite baking localhost into production bundle · Gemini JSON parse failures on malformed responses

✓ How We Fixed Them

app.use() instead of app.get("*") · Removed PORT from deploy vars · Conditional ADC fallback in firebase.js and storageService.js · .env.production with VITE_API_URL=/api · Regex JSON extraction from Gemini response

🚀 How We'd Improve It

Add progress tracking over multiple sessions with trend charts · Support real-time AI feedback during recording · Add Instagram OAuth to analyze public speaking clips · Multi-language coaching reports

📚 What We Learned

Chained AI prompting (Pass 1 → Pass 2) dramatically improves output quality · ADC is the right credential strategy for Cloud Run · GA4 Measurement Protocol enables server-side analytics without a browser · Vite env vars must be set at build time, not runtime
